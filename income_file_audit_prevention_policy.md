# Income File Audit Prevention Policy

## Objective
Audit every income/renda-related file with already-listed policy tools and convert weak content into prevention-focused, measurable, population-impact actions.

## Policy tools (already listed and reused)
- `Prompt_Income_IDH_Guard_Bundle`
- `Politica_Leitura_Pagamento_Propostas_Bundle`
- `incoming_income_audit_form.md`
- `populational_issue_prevention_idea_form.md`

## Mandatory rule
Read the full report before any interpersonal approach.

## Classification logic
1. `APPROVE_TRACK`
- File already includes prevention logic, control/audit signal, and measurable indicators.

2. `TRANSFORM_TO_PREVENTION`
- File has useful social/public-benefit intent but lacks prevention structure or measurable implementation.

3. `BLOCK_REVIEW`
- File indicates fraud, abuse, manipulation, violence, or institutional harm patterns.

4. `NEEDS_REWRITE`
- File is too vague or disconnected from population-impact outcomes.

## Prevention transformation standard
Every transformed file must include:
- target population
- preventable risk
- preventive action
- 90-day pilot scope
- measurable metric
- required evidence for audit
- safe/legal operational gate

## Population issue idea pipeline
Use `reports/populational_issue_idea_queue.md` to collect candidates from transformed files.
Then complete each candidate using:
- `populational_issue_prevention_idea_form.md`

## Expected outcome
- less reactive crisis response
- more preventive design
- clearer evidence trail
- more practical ideas to resolve population issues with measurable impact
